## 一、系统管理员默认账号

### 当前状态

种子数据 `db/seeds/002_users.sql` 中创建了示例用户，`004_user_role_assignments.sql` 分配了角色。但当前：

1. **默认密码为弱密码**：种子数据中的密码哈希对应的原始密码（如 `admin123`、`test123`）未在任何文档中明确标注
2. **首次部署无强制改密**：用户首次登录后不会被提示修改密码
3. **系统管理员角色绑定**：`system_admin` 角色的用户 `tenant_id` 为 NULL，但 `TenantContext` 中间件要求上下文中有 tenant_id
4. **缺少账号初始化脚本**：生产环境部署时应有独立的「创建初始管理员」流程

### 待改进

- [ ] **文档记录默认账号**：明确标注种子数据中的用户名和默认密码
- [ ] **首次登录强制改密**：在 `users` 表增加 `must_change_password` 标志位，或利用现有 `password_changed_at` 与 `created_at` 比较
- [ ] **生产环境初始化脚本**：提供独立脚本或 CLI 工具，用于创建初始系统管理员，并强制设置强密码
- [ ] **字段 `login_fail_count` 重置时机**：当前登录成功后未清零，需检查 `auth_service.go` 逻辑

## 二、提示词模板必须固定格式

### 当前状态

系统提示词模板使用 `{{variable}}` 格式的模板变量（如 `{{main_table}}`、`{{rules}}`），前端根据这些变量渲染配置页面和审核面板。

**问题**：
1. 提取阶段的提示词要求 LLM 输出固定 JSON Schema，前端根据这个 Schema 渲染审核结果面板。如果管理员修改了提示词中的 JSON Schema（如改字段名、增减字段），前端将无法正确解析和展示结果。

2. 推理阶段的用户提示词中的模板变量是硬编码的（`{{main_table}}`、`{{detail_tables}}`、`{{rules}}` 等），`prompt_builder.go` 中的 `strings.ReplaceAll` 也是硬编码匹配。如果模板变量名被修改，提示词构建将失败。

3. 当前提示词模板为只读（无编辑 API），但未来如果开放编辑，需要严格校验模板格式。

### 待改进

- [ ] **提示词模板格式校验**：定义必须包含的模板变量列表，编辑时校验
- [ ] **JSON Schema 固定化**：将提取阶段的 JSON 输出字段定义为常量，前后端共用
- [ ] **提示词版本管理**：支持提示词模板版本化，便于回滚
- [ ] **前端结果解析容错**：对 LLM 输出的 JSON 增加宽松解析（如正则提取、Markdown 代码块提取）
- [ ] **归档提示词区分**：归档提示词使用 `overall_compliance` 而非 `recommendation`，前端需区分两种结果结构

## 三、前端分页性能

### 当前状态

前端 `usePagination.ts` 实现了通用的前端分页工具，所有列表数据（审核日志、成员列表、配置列表等）采用**前端分页**策略：一次性从后端加载全部数据，前端做分页、搜索、排序。

**问题**：
- `settings.vue`（约 100KB）中审核工作台和归档复盘的字段列表数据量可能很大（数百个流程 × 数十个字段）
- 组织成员列表、用户配置列表在大型租户中可能有数千条记录
- 前端一次性加载全部数据会导致首次请求慢、内存占用高

### 待改进

- [ ] **后端分页**：在 Repository 层增加 `Paginate(page, pageSize int) ([]T, total int64, error)` 方法
- [ ] **前端适配**：修改 composables 中的 API 调用，传递 `page`/`pageSize` 参数，根据后端返回的 `total` 控制分页器
- [ ] **优先改造的接口**：
  - `GET /api/tenant/org/members` — 成员列表
  - `GET /api/tenant/user-configs` — 用户配置列表
  - `GET /api/tenant/settings/processes` — 个人流程列表
  - 日志查询类接口（audit_logs / cron_logs / archive_logs）
- [ ] **搜索后端化**：核心列表的关键字搜索移到后端（SQL LIKE / 全文检索）

## 四、国际化完善

### 当前状态

前端已有中文（`zh-CN.ts`，约 90KB）和英文（`en-US.ts`，约 90KB）语言包，覆盖了绝大部分 UI 文本。

**已发现的问题**：
1. 后端错误消息为硬编码中文（如 `"租户Token配额不足"`），不随用户语言偏好变化
2. 部分新增 UI 元素的 i18n key 可能未覆盖
3. 种子数据中的标签（如部门名称、角色名称）为中文，英文环境下显示中文

### 待改进

- [ ] **后端错误码国际化**：返回错误码而非中文消息，前端根据错误码查找国际化文本
- [ ] **种子数据双语**：考虑为种子数据添加 `_en` 后缀字段或使用 JSONB 存储多语言标签
- [ ] **定期检查 i18n 覆盖率**：编写脚本扫描 Vue 文件中的硬编码中文
