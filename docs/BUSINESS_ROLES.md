# 业务角色说明

> OA 智能审核平台 — 租户级业务角色定义

---

## 1. 概述

业务角色（Business Roles）是由租户管理员在「组织人员 → 角色权限」中定义和管理的角色。它们控制租户内用户可以访问哪些页面。

与系统角色（`system_admin` / `tenant_admin` / `business`）不同，业务角色是租户范围内的细粒度权限分配机制。

---

## 2. 默认业务角色

| 角色 ID | 角色名称 | 说明 | 默认页面权限 |
|---------|---------|------|-------------|
| ROLE-001 | 业务用户 | 普通业务人员，使用审核工作台等前台功能 | 仪表盘、审核工作台、个人设置 |
| ROLE-002 | 审计管理员 | 在业务用户基础上，额外拥有归档复盘和定时任务权限 | 仪表盘、审核工作台、定时任务、归档复盘、个人设置 |
| ROLE-003 | 租户管理员 | 可进入后台管理，配置规则、组织人员等 | 仪表盘、审核工作台、定时任务、归档复盘、个人设置、规则配置、组织人员、数据信息、用户偏好 |

以上三个角色为系统内置角色（`is_system: true`），不可删除，但可编辑其页面权限。

---

## 3. 可分配的页面

业务角色只能分配租户范围内的页面，不包含系统管理员页面：

| 页面路径 | 页面名称 | 分组 | 备注 |
|---------|---------|------|------|
| `/overview` | 仪表盘 | 通用 | 所有角色默认拥有，不可取消 |
| `/dashboard` | 审核工作台 | 业务功能 | |
| `/cron` | 定时任务 | 业务功能 | 依赖审核工作台 |
| `/archive` | 归档复盘 | 业务功能 | |
| `/settings` | 个人设置 | 通用 | 所有角色默认拥有，不可取消 |
| `/admin/tenant/rules` | 规则配置 | 后台管理 | |
| `/admin/tenant/org` | 组织人员 | 后台管理 | |
| `/admin/tenant/data` | 数据信息 | 后台管理 | |
| `/admin/tenant/user-configs` | 用户偏好 | 后台管理 | |

---

## 4. 依赖关系

- 定时任务（`/cron`）依赖审核工作台（`/dashboard`）：分配定时任务权限时，系统自动添加审核工作台权限；取消审核工作台时，自动取消定时任务。

---

## 5. 多角色支持

一个成员可以同时拥有多个业务角色。例如：
- 张明：业务用户 + 审计管理员
- 陈刚：业务用户 + 审计管理员 + 租户管理员

成员的实际页面权限为其所有业务角色的页面权限并集。

---

## 6. 与个人设置的联动

个人设置页面中的「审核工作台」「定时任务」「归档复盘」设置标签页，仅在以下条件同时满足时显示：
1. 当前激活的系统角色为 `business`（业务用户身份）
2. 当前租户下的业务角色包含对应页面权限

例如：用户切换到业务用户身份，且其业务角色包含 `/dashboard` 权限，才会看到「审核工作台」设置标签页。

---

## 7. 侧边栏菜单过滤

侧边栏的业务菜单区域（审核工作台、定时任务、归档复盘）会根据当前用户的业务角色 `page_permissions` 并集进行过滤，只显示用户有权限的菜单项。

例如：ROLE-001（业务用户）只有 `/dashboard` 权限，侧边栏只显示「审核工作台」，不显示「定时任务」和「归档复盘」。

实现位置：`composables/useSidebarMenu.ts`

---

## 8. 路由守卫

路由中间件（`middleware/auth.ts`）在系统角色权限检查之外，还会对 `business` 身份的用户进行业务角色页面权限检查。用户直接通过 URL 访问无权限的页面时，会被重定向到仪表盘（`/overview`）。

`/overview` 和 `/settings` 为所有角色默认可访问，不受此检查限制。
